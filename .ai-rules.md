
# AI Collaboration Rules - Explicit Consent Protocol

**Last Updated:** 2026-02-01  
**Purpose:** Enforce governance rules to prevent AI from making significant decisions without explicit human approval

---

## Core Principle

**AI MUST get explicit human approval before making ANY change that could impact system behavior, performance, or architecture.**

---

## Before Making ANY Change, AI Must:

1. ✅ **Check [`DECISION_AUTHORITY.md`](DECISION_AUTHORITY.md)** to determine the decision level (0-3)
2. ✅ **Describe the change** in plain, non-technical language
3. ✅ **Explain WHY** it's needed (what problem does it solve?)
4. ✅ **List alternatives** (minimum 2 options, preferably 3-4)
5. ✅ **State risks clearly** for each option
6. ✅ **Ask for explicit approval** using `ask_followup_question` tool
7. ✅ **Wait for human response** - NEVER assume approval

---

## Decision Level Quick Reference

### Level 0: Proceed Autonomously ✅
- Documentation updates
- Code comments
- Read-only operations (`read_file`, `list_files`, `search_files`)
- Creating test files
- Code formatting (no logic changes)

### Level 1: Must Ask First ⚠️
- Configuration values (thresholds, timeouts, weights)
- Dependencies (adding/updating packages)
- API changes (endpoints, schemas)
- Database schema changes
- Environment variables

### Level 2: Must Present Options 🔍
- Architecture decisions
- Algorithm changes
- Performance trade-offs
- Security-related changes
- Third-party integrations

### Level 3: Human-Only 🚫
- Production deployment
- Breaking changes
- Cost-impacting decisions
- Legal/compliance matters
- Business logic changes

**See [`DECISION_AUTHORITY.md`](DECISION_AUTHORITY.md) for complete details.**

## Forbidden Actions Without Approval

AI **CANNOT** do the following without explicit "yes" from human:

### ❌ Configuration Changes
- Changing thresholds (e.g., `CAUSE_THRESHOLD`, `LOCATION_THRESHOLD`)
- Modifying scoring weights
- Updating cache TTL values
- Changing API rate limits
- Altering timeout values
- Modifying retry counts

### ❌ Algorithm/Logic Changes
- Changing recommendation algorithms
- Modifying scoring logic
- Updating classification rules
- Altering ranking systems
- Changing filtering logic

### ❌ Data Structure Changes
- Making fields optional/required
- Changing data types
- Modifying API contracts
- Database schema changes
- Adding/removing model fields

### ❌ Dependency Changes
- Adding new npm packages
- Updating package versions
- Removing dependencies
- Changing package.json

### ❌ API Changes
- Adding new endpoints
- Modifying request/response schemas
- Changing HTTP methods
- Adding/removing parameters

### ❌ Deployment/Infrastructure
- Deploying to production
- Modifying CI/CD pipelines
- Changing deployment configs
- Environment variable changes

---

## Allowed Without Asking (Level 0 Only)

AI **CAN** do these autonomously:

### ✅ Documentation
- Adding/updating comments
- Creating/updating .md files
- Adding JSDoc/TSDoc
- Fixing typos in docs

### ✅ Read Operations
- Reading files
- Listing directories
- Searching code
- Analyzing code structure

### ✅ Test Creation
- Creating new test files
- Adding test cases
- Creating test fixtures

### ✅ Formatting
- Fixing indentation
- Organizing imports (no logic changes)
- Adding/removing whitespace

## Approval Process

### For Level 1 Changes (Quick Approval)

**AI must use this format:**

```markdown
I need to make a Level 1 change:

**What:** [Describe the change]
**Why:** [Explain the problem it solves]
**Impact:** [What will change in behavior]
**Risk:** [What could go wrong]

**Approve?** (yes/no)
```

**Example:**
```markdown
I need to make a Level 1 change:

**What:** Change CAUSE_THRESHOLD from 40 to 35
**Why:** Currently getting 0 recommendations for Nigeria church attack
**Impact:** Will match more organizations (lower bar for relevance)
**Risk:** May include less relevant organizations

**Approve?** (yes/no)
```

### For Level 2 Changes (Options Required)

**AI must use this format:**

```markdown
I need to make a Level 2 decision:

**Problem:** [Describe the issue]
**Impact:** [What's broken or suboptimal]

**Option 1:** [Description]
- Pros: [List benefits]
- Cons: [List drawbacks]
- Risk: [What could go wrong]

**Option 2:** [Description]
- Pros: [List benefits]
- Cons: [List drawbacks]
- Risk: [What could go wrong]

**Option 3:** [Description]
- Pros: [List benefits]
- Cons: [List drawbacks]
- Risk: [What could go wrong]

**Recommendation:** [Which option and why]

**Which option should I implement?**
```

**Example:**
```markdown
I need to make a Level 2 decision:

**Problem:** NTEE codes missing from Search API results
**Impact:** 0 recommendations for Nigeria church attack

**Option 1:** Lower CAUSE_THRESHOLD to 25
- Pros: Quick fix, no API changes
- Cons: May reduce precision, more false positives
- Risk: Lower quality recommendations

**Option 2:** Fetch from Details API for complete data
- Pros: Complete NTEE data, accurate scoring
- Cons: 10x slower, API rate limits
- Risk: Performance degradation, API costs

**Option 3:** Make NTEE scoring optional
- Pros: Balanced approach, works with incomplete data
- Cons: Inconsistent scoring across orgs
- Risk: Some orgs scored differently

**Option 4:** Remove NTEE scoring entirely
- Pros: Simple, no data dependency
- Cons: Lose valuable signal for relevance
- Risk: Less accurate recommendations

**Recommendation:** Option 3 - balanced approach

**Which option should I implement?**
```

## When AI is Unsure

If AI cannot determine the decision level:

1. **Default to Level 2** (present options)
2. Explain the uncertainty
3. Ask human to classify the decision
4. Wait for guidance

**Example:**
```markdown
I'm unsure if this is Level 1 or Level 2:

**Change:** Update caching strategy from memory to Redis
**Uncertainty:** Could be Level 1 (config) or Level 2 (architecture)

Should I:
A) Present options (treat as Level 2)
B) Just ask for approval (treat as Level 1)
C) Something else

Which approach?
```

---

## Emergency Override

Human can override any level with explicit instruction:

- **"Just do it"** → AI proceeds (Level 0 treatment)
- **"Show me options first"** → AI escalates to Level 2
- **"I'll handle it"** → AI stops and waits

---

## Enforcement Checklist

Before using `write_to_file`, `apply_diff`, or `insert_content`, AI must verify:

- [ ] Checked [`DECISION_AUTHORITY.md`](DECISION_AUTHORITY.md) for decision level
- [ ] If Level 1+: Used `ask_followup_question` to get approval
- [ ] If Level 2: Presented 2-4 options with pros/cons
- [ ] Received explicit "yes" or option selection from human
- [ ] Documented the decision in commit message

**If ANY checkbox is unchecked → STOP and get approval first**
